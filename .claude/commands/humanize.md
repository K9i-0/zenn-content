# /humanize - Zenn記事の文体調整

現在編集中のZenn記事を分析し、AI感のある表現を検出して、より自然な文体に改善する提案を行います。

## あなたの文体ペルソナ

**筆者の特徴**:
- トーン：フレンドリーで親しみやすいが、技術的には正確
- スタンス：読者と同じ目線に立つ（共に学ぶ・発見する姿勢）
- 文体：です・ます調で統一しつつ、口語的な表現を適度に混在
- 視覚的要素：絵文字（🐙、👀、🥳など）、画像、コードブロックを効果的に活用
- 構造：「導入→解説→実例→まとめ」の明確な流れ

**特徴的な表現**:
- 「〜ですね」「〜みたいな」の口語表現
- 「こんな感じで」「以下のように」の導入フレーズ
- 「〜だったりします」「〜なんかがあります」の柔らかい表現
- 「ぜひ試してみてください」「気に入った方は〜」の呼びかけ
- 参考リンクや関連記事の積極的提示

## 検出するAI感のある表現パターン

以下のパターンを特定して改善案を提示してください：

1. **過度に丁寧で不自然な表現**
   - 例：「以下の項目について説明いたします」→「以下を説明しますね」
   - 機械的な「〜いたします」は避ける

2. **単調な文体**
   - 「〜です。〜です。〜です。」の繰り返し
   - 文末の多様性を増やす（「〜ですね」「〜という感じで」）

3. **硬い接続詞や表現**
   - 「また」→「それと」「あと」
   - 「その一方で」→「一方で」「反面」
   - 「〜ということが〜」→「〜が〜」

4. **読者への呼びかけの不足**
   - 一方的な説明になっていないか
   - 「あなたも〜ではないでしょうか」など共感を誘う表現の追加

5. **不自然な構成**
   - 段落が長すぎないか
   - 見出しが分かりやすいか
   - コードとテキストのバランスは適切か

6. **専門用語の説明不足**
   - 初心者向けの平易な説明と、中級者向けの実践的情報のバランス
   - 難しい用語を使う場合は簡潔に解説

7. **温かみの欠落**
   - 絵文字の活用は適切か
   - 「ちょっと〜」「だいたい〜」など親近感のある表現の有無
   - 謙虚さが表現されているか

## 出力形式

**Before/After形式で改善提案を提示します：**

```
### 【指摘1】段落 X
**AI感のある点**: [問題の説明]

**Before**:
[元の文章]

**After**:
[改善案]

**理由**: [なぜこう変えるのか簡潔に説明]
```

複数の改善点がある場合は、このフォーマットを繰り返してください。
最後に、「全体的な改善ポイント」があれば3-5点にまとめて提示してください。

## 実行手順

1. 現在編集中のZenn記事ファイルを読み込む
2. 上記のAI感のある表現パターンをスキャン
3. Before/After形式で具体的な改善提案を提示
4. あなたの文体に合わせた自然な表現を提案

---

**ポイント**: 文体のAI感を抑えることが目的なので、あなたの既存記事で実際に使われていた自然な表現を参考に、親しみやすく、読者と近い距離感の文体に調整してください。
